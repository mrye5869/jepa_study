# LeWM 训练与推理全流程

---

## 一、训练流程：让模型学会"给定历史+动作 → 预测未来状态"

### 整体链路

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ ① 数据   │───▶│ ② 预处理 │───▶│ ③ 编码   │───▶│ ④ 切分   │───▶│ ⑤ 预测   │───▶│ ⑥ 投影   │───▶│ ⑦ Loss   │
│   加载   │    │          │    │          │    │ 上下文/  │    │          │    │   对齐   │    │   + 反向 │
└──────────┘    └──────────┘    └──────────┘    │  目标    │    └──────────┘    └──────────┘    │  传播   │
                                                └──────────┘                                    └──────────┘
```

### ① 数据加载

从 HDF5 文件读取预录专家演示轨迹，每次取连续的 `history_size + 1` 帧：

```
每个 batch（以 PushT 为例）:

  pixels: (128, 4, 3, 224, 224)    ← 连续 4 帧 (history=3 + 预测1步)
  action: (128, 3, 2)              ← 帧间的 3 个动作
```

### ② 预处理

```
pixels                                action
  │                                     │
  ▼                                     ▼
0→1 归一化                          Z-score 标准化
  │                                     │
  ▼                                     ▼
ImageNet 均值/标准差归一化            nan → 0 (序列边界)
  │
  ▼
Resize → 224×224
  │
  ▼
(B, T, C, H, W)                     (B, T-1, act_dim)
```

### ③ 编码（encode）

**图像编码：**

```
pixels (B, T, C, H, W)
  │
  ▼  flatten: (B,T,...) → (B×T, ...)    ← 关键：时间维拍平，每帧独立编码
  │
  ▼  ViT 逐帧编码
  │  取 [CLS] token → (B×T, 192)
  │
  ▼  projector MLP: 192 → 2048 → 192
  │  (BatchNorm + GELU)
  │
  ▼
emb (B, T, 192)                        ← 恢复时间维
```

**动作编码：**

```
action (B, T, act_dim)
  │
  ▼  permute: (B,T,D) → (B,D,T)
  │
  ▼  Conv1d(1×1) 逐时间步通道混合
  │
  ▼  permute: (B,D,T) → (B,T,D)
  │
  ▼  MLP: smoothed_dim → 4× → emb_dim
  │
  ▼
act_emb (B, T, 192)
```

### ④ 切分上下文与目标

```
emb (B, 4, 192)                    act_emb (B, 3, 192)
  │                                   │
  ├─ emb[:, :3]  → ctx_emb (帧0,1,2)  ├─ act_emb[:, :3] → ctx_act (动作0,1,2)
  │                                    │
  └─ emb[:, 1:]  → tgt_emb (帧1,2,3)
                    ↑
                    预测标签：预测器拿到帧 t 的嵌入 + 动作 t，
                    应预测帧 t+1 的嵌入
```

> 时序对齐示意：
>
> ```
> 帧:    [0]  ──动作0──▶  [1]  ──动作1──▶  [2]  ──动作2──▶  [3]
>        ctx               ctx              ctx              tgt
>                          tgt              tgt
> ```

### ⑤ 预测（ARPredictor）

```
ctx_emb (B, 3, 192)              ctx_act (B, 3, 192)
  │                                  │
  │  + 可学习位置编码                 │
  │                                  │
  ▼                                  ▼
┌──────────────────────────────────────────┐
│         ARPredictor (6层因果Transformer)   │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │   ConditionalBlock ×6            │    │
│  │                                  │    │
│  │   adaLN_modulation(c) ──────────┐│    │
│  │   生成 6组参数:                 ││    │
│  │   shift/scale/gate × 2(attn+mlp)││    │
│  │                                 ││    │
│  │   ┌─ attn ─┐   ┌─ mlp ─┐       ││    │
│  │   │因果掩码│   │       │       ││    │
│  │   └───┬────┘   └───┬───┘       ││    │
│  │      gate_msa       gate_mlp    ││    │
│  │       │               │         ││    │
│  │   x + gate*attn   x + gate*mlp  ││    │
│  │       │               │         ││    │
│  │      ◄──── modulate ──►         ││    │
│  └──────────────────────────────────┘    │
│                                          │
│  AdaLN-zero 核心:                        │
│  • gate 初始化为 0 → 训练初期条件被关闭   │
│  • 模型先学纯视觉预测，逐步学会用动作信号  │
│  • 因果掩码确保 t 时刻只能 attend ≤t       │
└──────────────────────────────────────────┘
  │
  ▼
pred_emb (B, 3, 192)
```

### ⑥ 投影对齐

```
pred_emb (B, 3, 192)
  │
  ▼  flatten: (B,3,192) → (B×3, 192)
  │
  ▼  pred_proj MLP: 192 → 2048 → 192
  │  (BatchNorm + GELU)
  │
  ▼  reshape: (B×3, 192) → (B, 3, 192)
  │
  ▼
pred_aligned (B, 3, 192)

为什么分开 projector 和 pred_proj？
即使输入输出维度相同(192→192)，这两个MLP作为
编码器空间 ↔ 预测器空间的适配层，去掉会损害收敛。
```

### ⑦ 计算 Loss + 反向传播

```
pred_aligned (B, 3, 192)          tgt_emb (B, 3, 192)
  │                                    │
  └────────── MSE(pred, tgt) ──────────┘
                    │
                    ▼
              pred_loss                   all_emb (B, 4, 192)
                    │                         │
                    │                         ▼
                    │                   SIGReg: Epps-Pulley检验
                    │                   投影到1024个随机方向
                    │                   检查分布是否≈N(0,I)
                    │                         │
                    │                         ▼
                    │                   sigreg_loss × 0.09
                    │                         │
                    └────────── + ────────────┘
                              │
                              ▼
                         total_loss
                              │
                              ▼
                    AdamW (lr=5e-5)
                    gradient_clip=1.0
                              │
                              ▼
                       更新所有参数
```

### SIGReg 细节

```
            嵌入 (T, B, 192)
                │
                ▼
     投影到 1024 个随机方向
     A = randn(192, 1024), 归一化
                │
                ▼
     每个方向 × 17 个检验点 t ∈ [0,3]
     cos(t · proj) 的经验均值 ≈ exp(-t²/2) ？
     sin(t · proj) 的经验均值 ≈ 0          ？
                │
                ▼
     Epps-Pulley 统计量 → 加权积分 → 平均所有方向
                │
                ▼
          sigreg_loss

     权重 0.09：
     • 太大 → 强制高斯过于刚性，丧失语义结构
     • 太小 → 坍缩拦不住，编码器输出常数向量
     • 0.09 = PushT/Cube/TwoRoom/Reacher 四任务上的均衡点
```

## 二、推理流程（MPC 规划）

模型在这里的角色是"世界模拟器"——不输出动作，只评估"如果执行这串动作，未来状态会怎样"。

### 整体循环

```
                    ┌──────────────────────────────────────────────┐
                    │                                              │
                    ▼                                              │
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────┴───┐
│ ① 编码   │──▶│ ② CEM   │──▶│ ③ Rollout│──▶│ ④ 评估   │──▶│ ⑤ 执行   │
│ 当前+目标│   │   采样   │   │   推演   │   │   代价   │   │ 最优动作  │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
                                                    │              │
                                                    │              ▼
                                                    │     ┌──────────────┐
                                                    │     │ ⑥ 环境返回   │
                                                    │     │   新观测     │──▶ 回到 ①
                                                    │     └──────────────┘
                                                    │
                                                    │ (CEM迭代)
                                                    ▼
                                            保留 top-16 → 拟合分布
                                                │
                                                └── 回到 ② 重采样
```

### ① 编码当前帧与目标帧

```
当前观测图像                          目标图像
    │                                    │
    ▼                                    ▼
encode(obs)                          encode(goal)
    │                                    │
    ▼                                    ▼
emb_0 (1, 192)                      goal_emb (1, 192)

两者使用完全相同的编码器（训练好的 ViT + projector）
```

### ② CEM 采样候选动作序列

```
第 1 轮：随机采样
─────────────────

    初始分布: N(0, I)
         │
         ▼
    采样 64 条候选动作序列
    每条 (T=16, action_dim)
         │
         ▼
    (B, 64, 16, 2)
```

```
第 N 轮：迭代精化
─────────────────

    上一轮 top-16 的动作序列
         │
         ▼
    拟合高斯分布 (均值 + 协方差)
         │
         ▼
    从新分布中重采样 64 条
         │
         ▼
    (B, 64, 16, 2)
```

### ③ Rollout：在嵌入空间自回归推演

> 核心：完全不碰新图像，纯在 192 维嵌入空间里滚

```
候选动作序列 (B, 64, 16, 2)
         │
         ▼  复制 64 份初始嵌入
emb = emb_0.expand(B, 64, -1, -1)    → (B, 64, 1, 192)
         │
         ▼  拍平 (B,64) → (B×64)，每条候选独立推演
emb (BS, 1, 192)        act (BS, H, 2)       act_future (BS, T-H, 2)
         │                    │                      │
         └────────────────────┼──────────────────────┘
                              │
              ┌───────────────▼───────────────┐
              │   自回归推演循环 (T-H 步)       │
              │                               │
              │  for t in range(n_steps):     │
              │                               │
              │  ① 编码当前动作                │
              │     action_encoder(act)        │
              │                               │
              │  ② 截取最近 HS=3 帧            │
              │     emb[:, -3:]  (BS, 3, 192) │
              │     act_emb[:, -3:]            │
              │                               │
              │  ③ 预测下一步嵌入              │
              │     ARPredictor → [:, -1:]     │
              │     (BS, 1, 192)              │
              │                               │
              │  ④ 拼回历史                    │
              │     emb = [emb, next_emb]     │
              │     act = [act, next_act]     │
              │                               │
              └───────────────┬───────────────┘
                              │
                              ▼
            predicted_emb (B, 64, T, 192)
            每条候选序列对应一条完整的嵌入推演轨迹
```

**Rollout 单步展开：**

```
t=0:    emb = [e₀]                              ← 初始嵌入
                  act_emb = encode([a₀, a₁, a₂])
                  ARPredictor([e₀], [a₀,a₁,a₂]编码) → ê₁

t=1:    emb = [e₀, ê₁]
                  act_emb = encode([a₀, a₁, a₂, â₀])
                              ↑ 注意：未来动作 â₀ 来自候选序列

t=2:    emb = [e₀, ê₁, ê₂]
                  ARPredictor([e₀,ê₁,ê₂], ...) → ê₃

...推演完所有未来步...

t=T:    emb = [e₀, ê₁, ê₂, ..., ê_T]
```

### ④ 计算代价

```
predicted_emb (B, 64, T, 192)          goal_emb (B, 64, T, 192)
         │                                        │
         │  只比较最后一步的嵌入                   │  目标嵌入广播到所有序列
         │  predicted_emb[:, :, -1:, :]          │
         │                                        │
         └────────── MSE(...) ────────────────────┘
                        │
                        ▼
                cost (B, 64)  ← 每条候选序列一个代价值
                值越小 → 最终状态越接近目标
```

### ⑤ 执行最优动作

```
64 个 cost 值
      │
      ▼  argmin
代价最小的候选序列 → 取其第 1 个动作
      │
      ▼
环境执行该动作 → 返回新观测
      │
      ▼
  回到 ① 重新规划（MPC 滚动优化）
```

## 三、核心设计决策速览

```
┌──────────────────────────────────────────────────────────────┐
│  设计                          │  为什么                    │
├──────────────────────────────────────────────────────────────┤
│  AdaLN-zero 条件注入           │  动作通过调制注入而非拼接   │
│  gate 初始化为 0               │  模型先学视觉，再学动作     │
├──────────────────────────────────────────────────────────────┤
│  分开的 projector / pred_proj  │  编码器↔预测器的空间适配   │
│  即使 192→192 维度也不合并     │  去掉会损害收敛             │
├──────────────────────────────────────────────────────────────┤
│  SIGReg (权重 0.09)            │  唯一防坍缩机制             │
│  强制嵌入分布 → N(0,I)        │  权重过大/过小都会失效      │
├──────────────────────────────────────────────────────────────┤
│  无 EMA, 无 stop-gradient      │  极简配方                   │
│  无辅助 loss                   │  2 个 loss 训到底           │
├──────────────────────────────────────────────────────────────┤
│  Rollout 在嵌入空间            │  不碰图像 → 快 48 倍        │
│  自回归推演                    │  192 维向量而非像素         │
└──────────────────────────────────────────────────────────────┘
```

## 四、模型角色总结

```
训练时：                   推理时：

输入 → 模型 → 输出          输入 → 模型 → 输出
图像+动作 → JEPA → 预测嵌入   图像+候选动作 → JEPA → 预测嵌入

模型学的是：                模型做的是：
P(下一帧嵌入 | 历史嵌入, 动作)   评估"这串动作会到达哪里"

                        动作选择由外部 CEM 完成：
                        采样 → 模型打分 → 保留top → 重采样 → 执行最优
```

**一句话：LeWM 不是策略模型，是世界模型。它不输出动作，只回答"如果我这么动，世界会变成什么样"。**
